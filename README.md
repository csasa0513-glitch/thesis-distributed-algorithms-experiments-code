# Experiments for Chapter 6

This folder contains the Python code that produces every figure and table
in Chapter 6 of the thesis *"Distributed Algorithms on Complex Networks"*.

## Layout

```
thesis-code/
├── README.md                  # this file
├── requirements.txt           # pinned dependencies
├── config.py                  # all experimental parameters (N, L, seeds, WS_P, ...)
├── games/
│   └── nash_cournot.py        # Cournot game: F_i, projections, solve_NE
├── graphs/
│   └── generators.py          # graph generators (cycle, wheel, grid, complete,
│                              #   watts_strogatz) + Koshal-style weight matrix W
├── algorithms/
│   ├── sync_koshal.py         # Algorithm 1 (synchronous)
│   └── async_gossip.py        # Algorithm 2 (asynchronous gossip)
├── analysis/
│   ├── metrics.py             # spectral indicators (rho_sync, rho_async),
│   │                          #   clustering C(G), path length L(G)
│   └── plots.py               # matplotlib helpers (mean +/- 90% CI band)
├── compute_NE_reference.py    # precompute the shared NE (results/NE_N{N}.npz)
├── run_Table1.py              # Table 1
├── run_table2_table3.py       # Tables 2 and 3
├── run_table4_table5.py       # Tables 4 and 5
├── run_async_table6.py        # Table 6
├── run_Table7.py              # Table 7
├── run_sync_p01.py            # Table 8 + Figure 3 (sync side)
├── run_async_p01.py           # Table 8 + Figure 3 (async side)
├── run_sync_sensitivity.py    # Table 9 + Figure 4 (sync side)
├── run_async_sensitivity.py   # Table 9 + Figure 4 (async side)
└── results/                   # csv / png / npz outputs (regenerated by the scripts)
```

`results/` is regenerated by the scripts and is intentionally excluded from
version control (see `.gitignore`); only the code is tracked.

## How to reproduce every figure and table

Install dependencies once:

```bash
pip install -r requirements.txt
```

(Optional) precompute the shared Nash equilibrium so that the paired
sync/async scripts read the same reference point from disk:

```bash
python compute_NE_reference.py
```

Then run the scripts:

```bash
python run_Table1.py            # Table 1: centralized sync baseline (complete graph)
python run_table2_table3.py     # Tables 2 & 3: synchronous algorithm on the 4 graphs
python run_table4_table5.py     # Tables 4 & 5: asynchronous gossip on the 4 graphs
python run_async_table6.py      # Table 6: gossip events to concurrence
python run_Table7.py            # Table 7: Watts-Strogatz C(G) and L(G)
python run_sync_p01.py          # Table 8 + Figure 3 (sync side)
python run_async_p01.py         # Table 8 + Figure 3 (async side)
python run_sync_sensitivity.py  # Table 9 + Figure 4 (sync side)
python run_async_sensitivity.py # Table 9 + Figure 4 (async side)
```

Total runtime is roughly 10-15 hours on a standard laptop (the sensitivity
sweeps dominate).

## Reproducibility

* All random draws are seeded from `config.SEED` (default `20260327`).
* The Nash-Cournot coefficients are drawn once in `config.py` and kept fixed
  across graphs; `config.resample(N)` redraws them for a different number of
  players.
* The synchronous and asynchronous scripts use `R = 50` sample paths per cell.
  The paired comparisons (Table 8 / Figure 3 and Table 9 / Figure 4) share the
  same graph realizations, the same initial points, and the same reference
  Nash equilibrium across sync and async; each script logs a sha256 fingerprint
  of `x*` so the pairing can be verified.
* The initial point is `xi_i ~ U([0, 10])^{2L}` projected onto the feasible
  set `X_i` (`xi` in the code corresponds to the thesis symbol `ξ_i`).
* The asynchronous stepsize uses the tuned diminishing rule `alpha_{k,i} = 9 / Γ_i^k`;
  the synchronous stepsize is `alpha_k = 1/k`. The constant-stepsize variant of
  Koshal (2016, §4.3) is out of scope.
* The full sweep `p_rew ∈ {0, 0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0}` feeds
  Table 9; Figure 4 plots the five values `{0.001, 0.01, 0.1, 0.5, 1.0}` for
  readability.
